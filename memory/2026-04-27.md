## Spawn Bug Fix - v5.2.2 (2026-04-27 10:00 UTC)

**Critical bug**: Deployed v5.2.1 was unplayable - no obstacles or coins spawning

**Root cause**: Initial spawn timing in `onMounted` handler was missing the 0.1s buffer that `resetStage()` uses:
- `onMounted`: `lastSpawnTime = clock.getElapsedTime() - spawnInterval` ❌
- `resetStage`: `lastSpawnTime = clock.getElapsedTime() - spawnInterval - 0.1` ✅

Without the buffer, `timeSinceLastSpawn` equals `spawnInterval` exactly on first frame, so `timeSinceLastSpawn > spawnInterval` is **false** and nothing spawns.

**Fix**: Added `- 0.1` buffer to `onMounted` to match `resetStage()` behavior

**Version bump**: v5.2.1 → v5.2.2 (required for deployment validation)

**Commit**: `54f5808` - fix: Add 0.1s spawn buffer in onMounted to match resetStage behavior

**Lesson**: Spawn timing initialization must be consistent between game start and stage resets. Always include the buffer to ensure immediate spawn on first frame.

## Spawn Bug Fix - v5.2.4 (2026-04-27 13:00 UTC)

**Issue**: CI tests still failing - obstacles/coins not spawning after debug stage jumps

**Root cause**: Initial game load creates a pending `setTimeout` callback (~4.5s total delay for 3-2-1-GO countdown). When test enters debug mode and presses stage key:
1. `resetStage()` correctly sets `lastSpawnTime` with -0.1 buffer ✅
2. **BUT** pending initial countdown timeout fires ~500ms later
3. It **overwrites** `lastSpawnTime` WITHOUT the -0.1 buffer ❌
4. Spawning breaks: `timeSinceLastSpawn == spawnInterval` exactly, so condition fails

**Fix** (commit `fa483e8`):
- Added `let initialCountdownTimeout = null;` to track the timeout
- Saved timeout reference: `initialCountdownTimeout = setTimeout(...)`
- In `resetStage()`: `if (initialCountdownTimeout) clearTimeout(initialCountdownTimeout);`
- Version bump: v5.2.3 → v5.2.4

**Commit**: `fa483e8` - fix: Clear pending initial countdown timeout in resetStage

**Lesson**: Pending async callbacks can overwrite state set by reset functions. Track and clear timeouts when resetting game state.
