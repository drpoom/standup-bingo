### Investigation Journey

- **Byte's browser debugging:** 3 failures — browser automation unreliable for gameplay testing

### Root Cause #1: `loadFachwerk is not defined` (FIXED in v4.5.7)
**Symptom:** Console showed `Uncaught ReferenceError: loadFachwerk is not defined`

**Why it blocked spawns:**
1. Function called at line 231 in `applyStageVisuals()`
2. Function defined at line 1179 (AFTER first use)
3. Crash during stage transitions prevented setTimeout callback from firing
4. `gameDuration` stayed at 0 forever
5. `spawnGraceActive = true` forever (0 < 1.5)
6. All spawns blocked

**Fix:** Moved `loadFachwerk` function from line 1179 to line 437 (before first use)
**Commit:** `847ef88`

### Root Cause #2: `getSurfaceZ is not defined` (FIXED in v4.5.8)
**Symptom:** Same type of bug — function called but never defined

**Locations:** Lines 1229, 1285, 3135, 3153

**Fix:** Replaced `getSurfaceZ()` calls with direct Z values (road curve only affects X/Y, not Z)
**Commit:** `dee6640`

### Key Lesson
**Function definition order matters in JavaScript!** Always define functions before first use, or use function declarations (hoisted) instead of arrow functions assigned to const.

## 🔧 Technical Notes

### JavaScript Function Hoisting
- Function declarations (`function foo() {}`) are hoisted
- Function expressions (`const foo = () => {}`) are NOT hoisted
- **Lesson:** Define before use, or use declarations for module-scope utilities

### Road Curve Implementation
- `getSurfaceY(z)` — applies vertical offset based on planet curvature
- `getSurfaceX(z)` — applies lateral offset (not used)
- `getCurveX(z)` — returns lateral curve offset for obstacles/buildings
- `getSurfaceZ(z)` — DOES NOT EXIST (Z is depth, no transformation needed)

### Spawn Logic Flow (for future reference)
```
resetStage() → gameDuration=0, lastSpawnTime=-2
↓
3-2-1-GO countdown (3s, countdownLocked=true)
↓
setTimeout callback (500ms delay)
  → stageTransitioning=false
  → gameDuration=1.5
  → lastSpawnTime=clock.getElapsedTime() - spawnInterval
↓
willSpawn check:
  - !spawnGraceActive (gameDuration >= 1.5) ✓
  - (time - lastSpawnTime) > spawnInterval ✓
  - !bonusNoSpawn ✓
  - !bossActive ✓
  - !stageTransitioning ✓
→ Spawns!
```
