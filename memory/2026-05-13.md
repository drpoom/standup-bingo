## Boss HP Bug — Root Cause & Fix

**Location:** `elango-surfers/src/App.vue`

**Problem:**
- Line 3537: `bossHealth.value = Math.min(250, ...)` → Sets 250 HP
- Line 3545: Timer decrements `(100 / 25) * realDelta` = 4 HP/sec
- Line 21: Health bar shows `bossHealth + '%'` → Displays 250% width

**Mismatch:** Health stored as absolute (250), displayed as percentage (250%), timer assumes 100.

**Fix (Option A — percentage-based):**
```javascript
// Line 3537
bossHealth.value = 100

// Line 3545 stays as-is (already correct for percentage)
```

## Stage 3 Skyscraper Bug — Root Cause Found

**Location:** `elango-surfers/src/App.vue` lines 2060-2080, 510-560

**Problem:** Buildings positioned with CENTER at ground level instead of BOTTOM.

```javascript
// Line 2070-2075 (regular buildings)
buildingGroup.position.set(
  side * (10 + Math.random() * 10),
  height / 2 + getSurfaceY(bldgZ),  // ← BUG: adds height/2
  bldgZ
);

// Line 520 (skyscrapers)
building.position.set(0, height / 2, 0);  // ← BUG: center, not bottom
```

**Effect:** Buildings float at `height/2` above ground (skyscrapers ~50-100 units high = very visible gap).

**Fix:**
```javascript
// Change to: position.y = getSurfaceY(bldgZ) only
// Remove the "height / 2 +" offset
```

**Status:** Investigation complete. Ready for Archie → Byte → Scout pipeline.
