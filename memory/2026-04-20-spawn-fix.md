assistant: I've fixed the spawning position for buildings and trees to correctly compensate for the curved ground. 🐕⚙️

**The Problem:**
Buildings and trees were being spawned using a linear Z-axis value. Because the game uses a "curved earth" model (where the road drops away as it recedes), objects spawned at a linear Z distance appeared to "float" or be misplaced relative to the actual curved surface of the world.

**The Fix:**
1. **Added `getSurfaceZ`**: I implemented a new helper function in `useCurve.js` that calculates the actual Z-coordinate on the sphere's surface. Instead of a straight line, it uses the sine of the angle relative to the planet's radius (`EARTH_R`), ensuring the Z-depth matches the curvature.
2. **Updated Spawning Logic**: I updated the initial spawn and the "recycle" logic for both trees and buildings in `App.vue` to use `getSurfaceZ()` instead of raw random Z values.
3. **Maintained Consistency**: The Y-axis compensation already existed, so the objects now align perfectly on both axes.
