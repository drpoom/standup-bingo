# Memory Flush — 2026-04-24

## Elango Surfers v5.1.10 Deployment (2026-04-23)

### Bugs Fixed
1. **Swipe controls broken** — `touchmove` listener was inline function preventing default but not tracking swipe gestures. Fix: Extract `handleTouchMove` as named function, register properly so `handleTouchEnd` swipe detection works.

2. **Grass texture invisible** — Mesh created but not visible. Root causes:
   - Position `y: -0.1` slightly above road level
   - No explicit render order (grass rendered after road, occluded)
   - Material lacked explicit `transparent/opacity` props
   - Fix: `y: -0.15`, `renderOrder: -1`, explicit material props

### Deployment Learnings
- **Git workspace corruption** — Multiple `.git/index.lock` files from crashed processes. `git checkout` fails with "untracked files would be overwritten". 
  - **Workaround:** Fresh clone from GitHub to `/tmp`, apply fixes, rebuild, deploy from clean clone.

- **GitHub Pages build not triggered** — Manual push to `gh-pages` branch does NOT trigger Pages build. Pages builds only when:
  1. GitHub Actions workflow runs (triggered by push to `main`)
  2. Workflow explicitly deploys to gh-pages
  - **Solution:** Push fix commit to `main`, let Actions workflow handle deployment.

- **Vite bundle hash stability** — Minor code changes (function extraction, material props) may not change bundle hash. Hash `index-DFGbwRCO.js` persisted across v5.1.9 → v5.1.10. Don't rely on hash change to verify deployment — check source code or runtime behavior.

```bash
npm install
npm run build
# Deploy via GitHub Actions (push to main)
```

| **405 SFX errors** | Missing `.ogg` files (`sfx-crash-wood.ogg`, `sfx-crash-glass.ogg`, `sfx-crash-metal.ogg`, `sfx-assembly.ogg`, `sfx-portal-whoosh.ogg`) | Fallback to existing SFX in `useAudio.js` |

| **Touch/swipe not working** | **Variable shadowing bug** in `handleTouchEnd()` | Fixed local `const` declarations shadowing refs |

## Critical Bug: Variable Shadowing

**Location**: `src/App.vue` line 2661-2662

**Before (broken)**:
```javascript
const touchEndX = e.changedTouches[0].clientX;  // ❌ Shadows ref
const diffX = touchEndX - touchStartX;  // ❌ ref - number = NaN
```

**After (fixed)**:
```javascript
const endX = e.changedTouches[0].clientX;  // ✅ Local variable
touchEndX.value = endX;  // ✅ Update ref
const diffX = touchEndX.value - touchStartX.value;  // ✅ number - number
```

**Lesson**: Never declare `const`/`let` with same name as a ref in the same scope — it shadows the ref and breaks reactivity.

## Technical Lessons

1. **Composable guards**: Always check `if (!scene) return;` at start of functions that depend on scene being initialized
2. **Ref shadowing**: Vue refs can be silently shadowed by local variables — use different names for locals
3. **Debug overlay value**: Real-time visibility into input handlers ends guesswork for Android-specific issues

- `src/composables/useScene.js` — Scene initialization guards
- `src/composables/useObstacles.js` — Scene guards

## Protocol Lessons Learned
- **Mickey = Router/Orchestrator ONLY** — do not implement except trivial 1-2 turn tasks
- **Complex work**: Spawn Archie (architecture) → Byte (code) → Scout (verify)
- **Creative work**: Spawn Sashay (SFX, visuals, copy)
- **Violations today**: Generated SFX directly (should be Sashay), fixed buildingTextures directly (should be Archie→Byte)

## Key Technical Lessons
- Module scope vs function scope: Variables needed across functions must be declared at module level (after imports, before functions)
- OGG 404 errors: Create actual audio files, not silent stubs — browsers may still report errors for empty/silent files
- State machine gating: Use `bossType === 'truck'` condition to differentiate truck vs dragon boss behavior
